## 3. 判定: 不採用

**9の法則は現行システムに対して-456,900円（-68.3%）の大幅悪化**をもたらす。

理由は明確で、現行のトレイリング利確（+1%→0.5%戻し）が「利を伸ばす」設計であるのに対し、9の法則は「9本で含み益があれば即利確」するため、**大きな利益を取り逃す**。

---

## 4. ルール別詳細分析

### 4.1 ルールA（9本含み益利確）

| 指標 | 値 |
|------|------|
| 発火回数 | 82回 |
| 総損益 | +456,800円 |
| 平均損益/回 | +5,571円 |
| 現行より改善した件数 | 34/82 (41.5%) |

ルールAは82回発火し、そのうち**41.5%（34件）で現行より良い結果**を出した。しかし残りの58.5%（48件）では「9本で利確したが、現行なら更に利益が伸びた」ケースであり、**逸失利益が改善分を大幅に上回る**。

> ルールAの問題: 含み益+0.1%の段階で利確してしまうが、現行のトレイリングなら+1.5%まで伸びるケースが多い。9本（=18分）は「利益が伸びる途中」であることが多い。

### 4.2 ルールB（5本膠着撤退）

| 指標 | 値 |
|------|------|
| 発火回数 | 168回 |
| 総損益 | -3,200円 |
| 平均損益/回 | -19円 |
| 現行より改善した件数 | 66/168 (39.3%) |

ルールBは168回発火（全取引の42%）と非常に高頻度。しかし**39.3%しか改善せず**、残りの60.7%は「膠着と見えたが、その後動き出して利益になった」ケースを取り逃している。

> ルールBの問題: 1分足で5本（=5分間）の膠着は日常的。±0.1%以内は「まだ動き出していない」だけで、その後の方向感が出る前に撤退してしまう。

### 4.3 合計影響

| 指標 | 値 |
|------|------|
| 9の法則で変更された取引 | 250/401件 (62.3%) |
| 変更されなかった取引 | 151件 |
| 全体への影響 | -456,900円 |

全取引の**62.3%が9の法則で早期エグジット**される。これは「ほぼ全ての取引を短期で打ち切る」に等しく、現行の「利を伸ばす」設計思想と根本的に矛盾する。

## 6. なぜ9の法則が現行に合わないか

### 6.1 現行の「利を伸ばす」設計との矛盾

現行のエグジット体系は以下の階段構造になっている:

```
含み益 +0.5% → 同値ストップ（損失ゼロ保証）
含み益 +1.0% → トレイリング開始（ピークから0.5%戻しで利確）
含み益 +1.5%以上 → トレイリングが追従（利益を最大化）
```

この設計は「小さな利益で満足せず、トレンドが続く限り保有する」思想。9の法則は「9本で含み益があれば即利確」なので、+0.3%程度の微益で利確してしまい、+1.5%まで伸びる機会を逃す。

### 6.2 時間軸の不一致

9の法則は「日足ベースの9日間」が原典。1分足で9本（=9分間）は、日足の9日間とは全く異なる意味を持つ。1分足の9本はノイズの範囲内であり、「トレンド消失」の判断には短すぎる。

### 6.3 現行の最大保有60本で十分

現行は既に`SHORT_MAX_HOLD_BARS = 60`（60分）で時間切れ決済を実装済み。これは「60分経っても利確に至らない＝トレンドが弱い」という合理的な判断であり、9本は早すぎる。

## 7. 代替案（本数ベースエグジットを改善する場合）

### 7.1 ルールAの改善

| 改善案 | 内容 | 期待効果 |
|--------|------|----------|
| **本数を増やす** | 9本→20〜30本 | トレイリングと競合しにくい |
| **含み益閾値を追加** | 含み益 > 0.5% かつ 20本以上 | 微益利確を防止 |
| **モメンタム減衰条件** | 直近3本のATRが縮小 | 「勢いが弱まった」時のみ発火 |

### 7.2 ルールBの改善

| 改善案 | 内容 | 期待効果 |
|--------|------|----------|
| **本数を増やす** | 5本→15〜20本 | 「本当の膠着」のみ検出 |
| **膠着幅を縮小** | ±0.1%→±0.05% | より厳密な膠着判定 |
| **出来高条件追加** | 出来高も減少 | 「枯れた」状態のみ撤退 |

### 7.3 推奨: 現行の60本ルールを段階化

現行の`SHORT_MAX_HOLD_BARS = 60`を以下のように段階化する方が、9の法則より合理的:

```
30本経過 + 含み益 < +0.3% → 同値ストップを+0.1%に引き上げ（微益確保）
45本経過 + 含み益 < +0.5% → 即時利確（伸び悩み）
60本経過 → 強制決済（現行通り）
```

これなら「利を伸ばす」設計を維持しつつ、「伸び悩み」を段階的に検出できる。

## 8. 結論

| 項目 | 判定 | 理由 |
|------|:---:|------|
| 9の法則（現仕様） | **不採用** | -456,900円(-68.3%)の大幅悪化。利を伸ばす設計と矛盾。 |
| ルールA単独 | **不採用** | 41.5%しか改善せず、逸失利益が大きい |
| ルールB単独 | **不採用** | 39.3%しか改善せず、ほぼ収支ゼロ |
| 段階的保有時間管理 | **検討価値あり** | 30本/45本/60本の段階化で再検証推奨 |

**現行のトレイリング利確+最大保有60本の組み合わせが最適**であり、9の法則の追加は不要。「含み益が伸び悩んだまま反転するケース」は、現行の同値ストップ（+0.5%で建値に引き上げ）で既にカバーされている。
