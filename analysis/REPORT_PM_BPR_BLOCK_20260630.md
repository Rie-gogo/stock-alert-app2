# 後場SHORT BPR>=0.65ブロック 検証レポート

**検証日:** 2026-06-30  
**検証期間:** 2026-06-17 〜 2026-06-30（10営業日）  
**ベース方式:** B2（前場のみブロック）+ 10銘柄  
**検証対象:** 後場（13:00以降）の全SHORTエントリーに対するBPR（Buy Pressure Ratio）フィルターの有効性

---

## 1. 検証目的

本番コード（realtimeSimEngine.ts）では、13:00以降の**全SHORT**エントリーに対してBPR>=0.65でブロックを適用している。一方、前回のバックテストでは**大台割れSHORTのみ**にBPRブロックを適用していた。この差異を明確にし、本番コードの設定が最適であるかを検証する。

BPR（Buy Pressure Ratio）は板情報から算出される買い圧力の比率であり、BPR>=0.65は「買い圧力が売り圧力を大幅に上回っている」状態を意味する。この状態でSHORTエントリーすると、買い圧力に押し戻されて損失になりやすいという仮説に基づくフィルターである。

---

| 区分 | 件数 | 損益 |

| 勝ち取引 | 13件 | +299,056円（逸失利益） |
| 負け取引 | 23件 | -227,934円（回避損失） |
| 引分取引 | 13件 | ±0円 |
| **差引効果** | **49件** | **+71,122円の損失回避** |

ブロックにより勝ち取引の利益も逸失しているが、負け取引の回避効果が上回り、ネットで+71,122円の改善効果がある。勝率は13/49 = 26.5%であり、全体平均（21.9%）よりやや高いものの、負け23件+引分13件 = 36件（73.5%）が非利益取引であることから、ブロックの正当性が確認できる。

### 最終判定

本番コードの設定（後場全SHORT BPR>=0.65ブロック）は、バックテストにより有効性が確認された。PF、総損益、期待値の全てが改善し、最大DDの悪化も10,286円と限定的である。閾値0.65は最適に近く、変更の必要はない。

### 補足: 大台割れのみブロック（パターンB）との比較

パターンBは大台割れSHORTのみ3件をブロックし、効果はほぼゼロ（パターンAとの差: -402円）。全SHORTに適用するパターンCの方が49件をブロックし、ネットで+25,409円の改善効果がある。大台割れに限定する理由はなく、全SHORTに適用すべきである。
