## 提案の概要

Phase 1〜4の段階的実装を提案。0.8%フィルター単独→データ蓄積→OOS検証→条件付き実装。

## 精査結果

### 賛同する点（技術的に正しい）

| 項目 | 評価 | 理由 |
|------|------|------|
| Phase分離（0.8%先行、v2後日） | **正しい** | 既に実装済み。効果の切り分けが可能 |
| OOS検証対象をSHORTのみに限定 | **正しい** | LONG PF0.50は明らかに不適格 |
| 仕様を事前固定しないとOOS汚染 | **正しい** | 後付け条件追加は検証の意味を失う |
| 最低成立数15件の要件 | **正しい** | 少数サンプルでのPF判定は無意味 |
| 不採用条件の明示 | **正しい** | 撤退基準がないと「もう少し調整すれば…」の罠にはまる |

### 懸念・修正が必要な点

| 項目 | 問題点 | 修正案 |
|------|--------|--------|
| 3ケース事前登録（V2-A/B/C） | 6920は既に取引除外済み。V2-Cは無意味 | V2-Cを削除し、V2-A/Bの2ケースに限定 |
| 「日次確認」で8項目集計 | 日次で手動確認は現実的でない。自動化すべき | 既存のrt_daily_reportに統合すれば自動化可能 |
| ブロックログの詳細フォーマット | 既にsignalHistoryに記録済み。別フォーマットは二重管理 | 既存signalHistory形式で十分 |
| 「保存すべきデータ」13項目 | 大半は既にrt_candles/rt_tradesに保存済み | 追加保存不要。既存DBで後日バックテスト可能 |
| SL 0.5% / TP 1.5%の固定 | 現行ロジックのSL/TPと異なる可能性。v2専用のSL/TPを意味するなら要確認 | v2のSL/TPは現行ロジックとは独立した「v2内部の判定基準」として理解 |
| 「スリッページ込みでPF≧1.20」 | スリッページの定義が曖昧。片道何円/何%を想定？ | 片道0.05%（≒25円@50000円銘柄）を標準とすべき |

### 過剰・不要な点

| 項目 | 理由 |
|------|------|
| 「Baseline仮想損益」の日次計算 | 0.8%フィルターなしの仮想取引を並行シミュレーションする必要がある。実装コストが高い割に、既存22日間のバックテストで既に差分は判明している |
| 「大台確認ステートの最終判定時刻」の保存 | signalHistoryに既に記録されている |
| 「通常ならエントリーした予定時刻/予定価格」の保存 | ブロック時のsignalHistoryレコードに含まれている |

## 現行システムとの整合性確認

### 既に満たしている要件

1. **1分足OHLCV保存** → `rt_candles`テーブルに全銘柄保存済み（今回の修正で除外銘柄も蓄積開始）
2. **大台シグナル成立時刻** → `signalHistory`に`round_break_detected`として記録済み
3. **0.8%ブロック判定** → `signalHistory`に`round_distance_block`として記録済み
4. **板情報・BPR** → `rt_candles`の`boardSnapshot`カラムに保存済み
5. **ブロックログ** → signalHistoryの`round_distance_block`エントリに日時・銘柄・方向・乖離率が含まれる

### 追加実装が必要な項目

**なし。** 現行システムで後日のCB v2バックテストに必要なデータは全て保存されています。

## 固定すべきCB v2 SHORT仕様（凍結版）

```
対象:       大台確認SHORT かつ 大台乖離率 > 0.8%
反発確認:   impulseLowから0.2%以上反発
戻り高値:   2本連続で高値を更新できない
戻り終了:   終値で5MAを上から下へ再クロス
再ブレイク: 終値 < impulseLow
エントリー: 条件成立後の次足始値
タイムアウト: 20本
SL:         0.5%
TP:         1.5%
方向:       SHORTのみ
```

### OOS検証ケース（事前登録）

```
Case V2-A: SHORT・終日・全アクティブ銘柄
Case V2-B: SHORT・11:30まで・全アクティブ銘柄
```

### OOS採用基準

```
CB成立数 >= 15件
CB部分PF >= 1.50
CB部分期待値 > 0
CB部分勝率 >= 35%
CB部分SL率 <= 60%
戦略全体損益 >= 0.8%単独
戦略全体PF >= 0.8%単独
最大DD <= 0.8%単独
1銘柄依存率 <= 50%
スリッページ込み(片道0.05%)PF >= 1.20
```

### 不採用条件

```
CB部分PF < 1.0
CB部分期待値 <= 0
CB成立数 < 15件での評価
戦略全体PFが0.8%単独を下回る
最大DDが悪化
1銘柄に50%以上集中
OOS後の条件変更が必要
```
