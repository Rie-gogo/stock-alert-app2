## 1. 背景と目的

前回の個別銘柄レジーム方式バックテストにおいて、パターンB（SHORT medium全解除）が最も高い総損益（+506,671円）とPF 1.17を記録した。しかし、上昇日に-240,043円の損失が発生し、最大DD 229,421円が許容範囲（200,000円）を超過していた。

本検証では、パターンBをベースに「上昇日フィルター」を追加する5つの方式を比較し、**反転耐性**（前場上昇→後場反転下落パターンへの対応力）を重視して最適方式を判定する。

## 3. 方向性判定ロジック

全銘柄の始値比変化率をリアルタイムで集計し、以下の条件で判定する:

> **bullish**: 上昇銘柄/下落銘柄 >= 2.0 or 中央値 >= +0.5% or 60%以上が+0.5%以上  
> **bearish**: 下落銘柄/上昇銘柄 >= 2.0 or 中央値 <= -0.5% or 60%以上が-0.5%以下  
> **neutral**: 上記いずれにも該当しない

## 14. 結論と推奨

### B2（前場のみブロック方式）を採用推奨

**採用理由:**
1. **最高PF 1.21** — 全5方式中最高。取引の質が最も高い
2. **最高総損益 +529,680円** — B0を23,009円上回る
3. **最高期待値 974円/回** — B0の793円/回から23%改善
4. **最良反転比率 0.24** — 上昇日の損失を最小限に抑制
5. **後場の反転利益を逃さない** — 6/24型で後場SHORTを完全獲得
6. **ロジックがシンプル** — 9:30時点の1回判定のみで実装容易

**残課題: 最大DD 229,421円**

全パターンで最大DDが229,421円と共通しており、これは6/30の前場損失（-170,888円）に起因する。B2のフィルターは6/30の9:30時点でneutral判定のため発動しない。この問題は上昇日フィルターでは解決できず、**前場全体のエントリー品質改善**（別課題）で対処すべき。

### B1/B4が不採用の理由

動的更新方式（B1）と閾値引き上げ方式（B4）は:
- 下落日の利益を-125,000円削減（B0比）
- PFが1.13に低下（基準1.15未満）
- bullish判定が「正しい上昇日」だけでなく「反転途中の一時的上昇」でもSHORTをブロックし、利益機会を逸失

### 実装仕様（B2方式）

```
エントリー時刻が前場（< 11:30）の場合:
  9:30時点の方向性判定を取得
  if (direction === "bullish" AND confidence === "medium" AND side === "short"):
    → ブロック
  else:
    → 許可

エントリー時刻が後場（>= 12:30）の場合:
  → 無条件で許可（SHORT medium含む）
```
