# CB v2 SHORT 凍結仕様（2026-07-16固定）

## シミュレーション対象条件

- 大台確認SHORTシグナルが発生
- 大台乖離率 > 0.8%（通常なら0.8%フィルターでブロックされるケース）

## CB v2 SHORTステートマシン

### ステート1: 反発確認待ち
- impulseLow（ブロック時の安値）から0.2%以上反発したら次へ
- タイムアウト: 20本でキャンセル

### ステート2: 戻り高値形成待ち
- 2本連続で高値を更新できなければ次へ
- タイムアウト: 20本（ステート1開始から通算）

### ステート3: 戻り終了確認
- 終値で5MAを上から下へ再クロスしたら次へ
- タイムアウト: 20本（通算）

### ステート4: 再ブレイク確認
- 終値 < impulseLow で成立→エントリー
- タイムアウト: 20本（通算）

## エントリー・決済条件

- エントリー: 条件成立後の次足始値
- SL: エントリー価格から+0.5%
- TP: エントリー価格から-1.5%
- 方向: SHORTのみ

## OOS検証ケース

- Case V2-A: SHORT・終日・全アクティブ銘柄
- Case V2-B: SHORT・11:30まで・全アクティブ銘柄

## 日次シミュレーション出力項目

- CB候補数（0.8%ブロックされたSHORTの数）
- CB成立数（全ステートクリアしてエントリーした数）
- タイムアウト数
- 勝ち数・負け数
- 総損益
- PF
- 各取引の詳細（銘柄・時刻・エントリー価格・決済価格・損益・決済理由）

## 5日間累計での採用基準

- CB成立数 >= 15件（5日間合計）
- CB部分PF >= 1.50
- CB部分期待値 > 0
- CB部分勝率 >= 35%

## 実装メモ

- 日次レポートハンドラ: server/scheduledHandlers.ts の rtDailyReportHandler
- 1分足データ取得: getRtCandlesAllForDate(date) from server/db.ts
- signalHistory取得: getSignalHistory() from server/realtimeSimEngine.ts
- 0.8%ブロックされたSHORTケースの特定: signalHistoryからaction='round_distance_block'かつside='short'を抽出
- 各ケースについて、ブロック時刻以降の1分足でCB v2ステートマシンを走らせる
