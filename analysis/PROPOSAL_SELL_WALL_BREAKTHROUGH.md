# 売り板突破パターン認識ロジック設計書

## 概要

現在のシステムは `sell_pressure` 時にLONGエントリーを一律禁止している。しかし6/30のように「売り板が厚いのに買いが突破する」パターン（機関投資家の買い集め）では、sell_pressure時こそLONGの好機となる。本提案は、このパターンをリアルタイムで識別し、sell_pressure時LONGブロックを条件付きで解除するロジックを設計する。

## 識別ロジック設計

### 関数名: `detectSellWallBreakthrough`

### 入力
- 直近N本のキャンドルバッファ（boardSnapshot含む）

### 判定条件（全て AND）

| # | 条件 | 閾値 | 根拠 |
|---|------|------|------|
| 1 | sell_pressure状態で連続陽線 | **3本以上** | 6/30の全パターンが3本以上 |
| 2 | BPR上昇トレンド | 直近3本BPR平均 > 開始時BPR + **0.05** | 6/30: 0.45→0.63(+0.18), 6/26: 0.48→0.44(-0.04) |
| 3 | 出来高維持 | 直近3本平均 >= 最初3本平均の **80%** | 6/30: 維持, 6/26: -70%減少 |
| 4 | 累積上昇率 | **>= 1.0%** | ノイズ除外。6/30は+5.27%, 6/26は+0.22% |
| 5 | 陽線の平均実体 | **>= 0.3%** | 6/30: 平均+1.04%, 6/26: +0.22% |

### 出力
- `detected: boolean` — パターン検出の有無
- `strength: number` — 強度スコア（0〜3）
- `startTime: string` — パターン開始時刻
- `cumulativeGain: number` — 累積上昇率

### 強度スコアの計算

```
strength = 0
if (連続陽線 >= 5本) strength += 1
if (BPR上昇 >= 0.10) strength += 1  
if (累積上昇率 >= 2.0%) strength += 1
```

## エンジンへの統合方法

### 方法A: sell_pressureブロック解除（推奨）

```typescript
// enterPosition() 内の既存ロジック
if (side === 'long' && boardSignal === 'sell_pressure') {
  // 現在: 一律ブロック
  // 提案: 売り板突破パターン検出時は解除
  const breakthrough = detectSellWallBreakthrough(buffer);
  if (!breakthrough.detected) {
    // 従来通りブロック
    return;
  }
  // 突破パターン検出 → ブロック解除してエントリー許可
  logSignal('sell_wall_breakthrough', breakthrough);
}
```

### 方法B: 板読みスコアへのボーナス加算

```typescript
// boardReadingScore() 内
function boardReadingScore(symbol, side, snapshot) {
  let score = /* 既存の7要素計算 */;
  
  // 売り板突破パターンボーナス
  if (side === 'long' && breakthrough.detected) {
    score += breakthrough.strength; // +1〜+3
  }
  
  return score;
}
```

### 方法C: A + B の併用（最も安全）

1. 突破パターン検出時のみ sell_pressure ブロックを解除
2. さらに板読みスコアにボーナスを加算
3. ただし最低でも strong 品質のシグナルが必要（medium は引き続きブロック）

## リスク管理

### 偽突破への対策

| リスク | 対策 |
|--------|------|
| 寄り付き直後のオーバーシュート | 09:00-09:10は除外（寄り付きの乱高下） |
| 一時的な反発後の再下落 | 累積上昇率1.0%の閾値で小さな反発を除外 |
| 出来高を伴わない上昇 | 出来高維持条件（80%以上）で薄商い上昇を除外 |
| BPRの一時的スパイク | 3本平均で平滑化 |

### 追加安全装置

1. **1日1回制限**: 同一銘柄で突破パターンLONGは1日1回まで
2. **損切り幅縮小**: 通常の-0.5%ではなく-0.3%に縮小（逆行時の損失を限定）
3. **利確幅拡大**: 突破パターンは大きく動くため、TP=+1.5%に拡大
4. **時間制限**: 前場のみ有効（後場は利益確定売りで反転リスク大）

## 補足: 6/23-6/24のsell_pressure分析

6/23と6/24は sell_pressure 時の陽線率が **0%** だった。これは「板も値動きも完全に売り一色」の日であり、突破パターンの条件（3本連続陽線）が物理的に成立しない。つまり下落日では自然にフィルターされる安全設計になっている。

| 6/24 | 733本 | **0%** | 不可能 |
| 6/25 | 999本 | 40% | 要検証 |
| 6/26 | 1,464本 | 44% | 要検証（累積率で除外される見込み） |
| 6/30 | 1,010本 | 43% | **検出対象** |
