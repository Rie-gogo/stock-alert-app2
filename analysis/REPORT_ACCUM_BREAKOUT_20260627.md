# 蓄積ブレイク（並び抜け）シグナル追加 バックテスト報告

**作成日**: 2026年6月27日  
**分析期間**: 2025年11月4日 〜 2026年6月12日（139営業日）  
**データソース**: J-Quants 1分足データ（17銘柄）  

---

## 1. 検証概要

「同じ価格帯で横ばい→放れが強力シグナル」という蓄積ブレイクを、現行シグナル群に追加した場合の損益影響を検証した。

### テスト仕様

| パラメータ | 設定値 |
|-----------|--------|
| 横ばい判定本数 (N) | 5本 |
| 横ばい閾値 (range_ratio) | ≤ 0.4% |
| ブレイク確認マージン | ±0.05% |
| 出来高倍率 | 直近N本平均 × 1.5 |
| ADXフィルター | ≥ 20 |
| レジームフィルター | 上抜け: レジーム≠down / 下抜け: レジーム≠up |
| ロット比率 | 30%（現行の既存シグナルとは独立） |
| 決済ロジック | 現行と同一（損切り/トレイリング/同値撤退/時間切れ/昼休み前/大引け） |

### 3.2 ロング/ショート別

| 方向 | 件数 | 勝率 | 損益 | 平均/件 |

上抜け（ロング）は若干プラスだが、下抜け（ショート）はマイナス。ショート方向の蓄積ブレイクは現行の下落トレンドフィルターと競合し、有効性が低い。

## 4. パラメータ感度分析

| 横ばい本数 (N) | シグナル発生数 | 1日平均 |
|:---:|:---:|:---:|
| N=3 | 28,925件 | 208件/日 |
| N=4 | 23,746件 | 171件/日 |
| N=5（採用値） | 19,816件 | 143件/日 |
| N=6 | 16,883件 | 121件/日 |

いずれのNでもシグナル発生数が極めて多い。1分足で「横ばい5本（=5分間）」は日常的に発生する現象であり、**フィルターとしての選別力が不十分**。

---

## 5. 問題点の整理

### 5.1 シグナル過多（ノイズ問題）

1分足での「5本横ばい→ブレイク」は、日常的な微小レンジの上下動を大量に拾ってしまう。1日24件のシグナルは実運用上も管理不能であり、バックテスト上も取引コスト（スリッページ・手数料）を考慮すると赤字に転落する可能性が高い。

### 5.2 リスク増大

最大DDが49,900円→124,350円（+149%）に悪化。最悪日も-30,900円→-65,200円に悪化。蓄積ブレイクの損切り頻発が原因。

### 5.3 エッジの希薄さ

平均損益+12円/件は、スリッページ（1ティック=数十〜数百円）を考慮すると実質マイナス。「横ばいからのブレイク」は1分足レベルでは統計的優位性を持たない。

## 6. 総合判定

| 項目 | 判定 | 理由 |
|------|:---:|------|
| 蓄積ブレイク（現仕様） | **不採用** | +39,750円(+5.9%)の微増だがDD+149%。リスク対リターンが悪い。 |
| 上抜けのみ採用 | **保留** | +79,400円だがDD悪化は残る。追加フィルターで改善の余地あり。 |
| 下抜け | **不採用** | -39,650円のマイナス。現行ショートシグナルと競合。 |

---

## 7. 改善提案（採用する場合）

蓄積ブレイクの概念自体は有効だが、**1分足での5本横ばいは条件が緩すぎる**。以下の改善で再検証する価値がある:

### 7.1 時間軸の変更

| 改善案 | 内容 | 期待効果 |
|--------|------|----------|
| **5分足ベース** | 5本=25分間の横ばい | シグナル数1/5、質向上 |
| **N=15〜20本** | 1分足15〜20本=15〜20分間 | より確実な蓄積を要求 |

### 7.2 追加フィルター

| フィルター | 条件 | 目的 |
|-----------|------|------|
| **出来高倍率引き上げ** | ×2.0〜2.5 | ダマシ排除 |
| **range_ratio引き下げ** | ≤0.2%（現行0.4%の半分） | より厳密な横ばい |
| **ブレイク幅拡大** | ×1.002（現行0.05%→0.2%） | 確実なブレイクのみ |
| **信頼度「強」限定** | 現行3要素で3点のみ | 高品質シグナルに限定 |
| **板読みスコア併用** | 板圧力がブレイク方向 | リアルタイム需給確認 |

### 7.3 推奨パラメータ（再検証候補）

```
N = 15（15分間の横ばい）
range_ratio ≤ 0.002（0.2%以内）
出来高 ≥ 平均 × 2.0
ブレイクマージン = 0.002（0.2%）
ADX ≥ 25
信頼度 ≥ medium
```

これにより「本当に蓄積された後の力強いブレイク」のみを捕捉し、日常的なノイズを排除できる可能性がある。

---

## 8. 結論

**現時点では蓄積ブレイクシグナルの追加は非推奨**。理由は:

1. **リスク対リターンが悪い**: +39,750円の利益増に対し、最大DD+74,450円の悪化
2. **エッジが希薄**: 平均+12円/件はスリッページで消滅する
3. **シグナル過多**: 1日24件は実運用不可能

ただし、「横ばい→放れ」の概念自体は有効であり、**パラメータを大幅に厳格化**（N=15、range≤0.2%、vol×2.0）した上で再検証する価値はある。特に5分足ベースでの検証を推奨する。
