## 4. ステートマシン動作分析

| 指標 | 値 |
|------|---:|
| SM発動（大台割れ検出） | 364件 |
| SM→エントリー | **1件（0.3%）** |
| SM→キャンセル | 363件（99.7%） |

### 問題: エントリー率が極端に低い（0.3%）

Ver2の条件が**厳しすぎる**ため、ほぼ全ての大台割れシグナルがキャンセルされている。結果として「大台割れを事実上無効化した」のと同等の効果になっている。

**キャンセル理由の推定**:
- 8本以内に「0.05%以上戻り → 陰線 + MA5以下」の2段階を完了するのが困難
- 特にエントリー価格制御（`entryPrice <= signalPrice`）が追加のハードルとして機能
- 大台割れ後は急落が続くことが多く、「戻り」自体が発生しにくい

## 7. 考察

### 7.1 なぜ効果があるか

大台割れシグナルは現行で**最大の損失源**（-51,400円、勝率27.8%）であり、これを事実上無効化することで:
- 損切り29回削減
- 最大DD 33.4%改善
- 総損益+27,200円改善

### 7.2 なぜ2基準FAILか

**エントリー数-23.7%**: 大台割れ72件のうち71件がキャンセルされ、他シグナルと合わせて-59件。15%上限を超過。

**大台割れ未プラス化**: 唯一エントリーした1件が-4,700円の損切り。ただし-51,400円→-4,700円は**90.9%の改善**。

### 7.3 実質的な意味

SM-v2は「大台割れシグナルを99.7%ブロックするフィルター」として機能している。これは:
- **大台割れシグナル自体を無効化する**のと同等
- ステートマシンの複雑な実装は不要で、単に大台割れシグナルを削除すれば同じ結果が得られる

---

## 8. 提案: 3つの選択肢

### 選択肢A: 大台割れシグナルを単純に無効化

ステートマシン不要。大台割れシグナルをコードから削除するだけで+46,700円の改善。最もシンプル。

### 選択肢B: SM-v2の条件を緩和してエントリー率を上げる

- 待機時間: 8本 → 15本に延長
- 戻り条件: 0.05% → 0.03%に緩和
- 戻り終了: 「陰線 + MA5以下」→「陰線のみ」に簡略化
- エントリー価格制御: 廃止（signalPriceより高くてもOK）

これにより大台割れの「良いエントリー」（現行20勝分）を一部回収できる可能性がある。

### 選択肢C: 大台割れに追加フィルターのみ適用

ステートマシンではなく、大台割れの**発生条件を厳格化**:
- 出来高 >= 平均の2倍（現行は0.8倍）
- 5本以上大台以下を維持
- ADX >= 25
- slope < -0.0002（明確な下落トレンド）
